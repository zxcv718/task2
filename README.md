# 나만의 퀴즈 게임

## 1. 프로젝트 개요

이 프로젝트는 Python으로 만든 콘솔 기반 퀴즈 게임입니다.  
사용자는 메뉴를 통해 퀴즈를 풀고, 새 퀴즈를 추가하고, 저장된 퀴즈 목록을 확인하고, 퀴즈를 삭제하고, 최고 점수와 플레이 기록을 확인할 수 있습니다.

프로그램 상태는 프로젝트 루트의 `state.json`에 저장됩니다.  
그래서 프로그램을 종료한 뒤 다시 실행해도 추가한 퀴즈, 최고 점수, 플레이 히스토리를 이어서 사용할 수 있습니다.

최근 리팩터링으로 게임 운영에 쓰이는 숫자 상수뿐 아니라 메뉴 문구, 입력 안내 메시지, `state.json` 키 이름까지 `game_constants.py`로 모아 관리하도록 정리했습니다.  
또한 `state_store.py`에서 `state.json`의 현재 스키마를 엄격하게 검증하고, 필수 키가 빠지거나 값 타입이 다르거나 JSON이 손상된 경우에는 전체 state를 기본 데이터로 복구하도록 구성했습니다.

---

## 2. 퀴즈 주제와 선정 이유

퀴즈 주제는 컴퓨터, Python, Git, 네트워크 기초입니다.

이 주제를 고른 이유는 다음과 같습니다.

- 미션의 핵심 학습 내용인 Python과 Git과 자연스럽게 연결됩니다.
- 문제를 직접 추가하면서도 주제를 확장하기 쉽습니다.
- 초심자 기준으로도 정답 여부를 이해하기 쉬운 객관식 문제를 만들기 좋습니다.

기본 퀴즈는 `default_data.py`의 `DEFAULT_QUIZZES`를 바탕으로 생성되며, 현재 8개의 기본 문제가 준비되어 있습니다.

## 4. 평가표 기준 기능 정리

### 4-1. 메뉴 출력과 기본 기능

프로그램을 실행하면 `QuizGame.run()`이 메뉴를 반복 출력합니다.  
현재 메뉴는 아래 6개 항목으로 구성되어 있습니다.

1. 퀴즈 풀기
2. 퀴즈 추가
3. 퀴즈 목록
4. 퀴즈 삭제
5. 점수 확인
6. 종료

평가표의 기본 요구 기능인 퀴즈 풀기, 퀴즈 추가, 퀴즈 목록, 점수 확인은 모두 포함되어 있습니다.  
추가로 보너스 기능인 퀴즈 삭제도 구현되어 있습니다.

### 4-2. 퀴즈 풀기와 정답 판정

퀴즈를 시작하면 먼저 몇 문제를 풀지 선택합니다.  
이후 선택된 수만큼 문제가 랜덤으로 출제됩니다.

정답 판정은 `Quiz.is_correct()`가 담당합니다.  
사용자가 입력한 번호와 정답 번호를 직접 비교하는 방식이라 판정 기준이 단순하고 명확합니다.

퀴즈 진행 중에는 힌트 기능도 사용할 수 있습니다.

- `h` 입력 시 현재 문제의 힌트를 보여 줍니다.
- 힌트를 사용하고 맞히면 `7점`
- 힌트 없이 맞히면 `10점`
- 틀리면 `0점`

### 4-3. 입력 검증

입력 검증은 `console_io.py`와 `quiz_session.py`에서 처리합니다.

현재 처리되는 입력 예외는 다음과 같습니다.

- 앞뒤 공백 제거
- 빈 입력 방지
- 숫자 변환 실패 처리
- 허용 범위 밖 숫자 처리
- 퀴즈 답 입력 시 `1`~`4` 또는 `h`만 허용

즉 평가표의 공통 입력 처리 기준에 맞춰, 잘못된 입력이 들어오면 안내 메시지를 보여 주고 다시 입력받도록 구현했습니다.

### 4-4. 안전 종료

입력 도중 `Ctrl+C` 또는 EOF가 발생하면 프로그램이 바로 비정상 종료되지 않도록 처리했습니다.

흐름은 다음과 같습니다.

- `prompt_text()`가 `KeyboardInterrupt`, `EOFError`를 감지
- `SafeExitRequest` 예외를 발생시켜 일반 입력 오류와 구분
- `run()`이 이를 받아 `safe_exit()` 실행
- 가능한 범위에서 `state.json` 저장 후 종료

또한 퀴즈를 푸는 도중 입력이 중단되면, 현재까지 진행한 결과를 `history`에 기록한 뒤 저장하도록 수정되어 있습니다.

### 4-5. 퀴즈 추가, 목록, 삭제

퀴즈 추가 기능에서는 다음 정보를 입력받습니다.

- 문제
- 선택지 4개
- 정답 번호
- 힌트

추가가 끝나면 새 퀴즈는 `state.json`에 즉시 저장됩니다.

퀴즈 목록 기능에서는 저장된 모든 퀴즈의 다음 정보를 확인할 수 있습니다.

- 화면용 문제 번호
- 문제
- 선택지

정답 번호와 힌트는 퀴즈 목록 화면에서는 숨기고, 실제 풀이 과정에서만 사용합니다. 이렇게 해야 목록 확인 기능이 정답 열람 기능처럼 보이지 않고, 퀴즈 게임의 성격도 더 잘 유지됩니다.

퀴즈 삭제 기능에서는 화면에 표시된 번호를 선택해 삭제할 수 있고, 삭제 결과도 즉시 `state.json`에 반영됩니다.

### 4-6. 최고 점수와 플레이 히스토리

퀴즈가 정상 종료되거나, 퀴즈 진행 중 입력이 중단되어도 현재까지의 결과가 `history`에 기록됩니다.

기록되는 값은 다음과 같습니다.

- `played_at`
- `selected_count`
- `answered_count`
- `correct_count`
- `score`
- `hint_used_count`

최고 점수는 `best_score`에 저장되며, `show_scores()`에서 최고 점수와 전체 플레이 횟수, 전체 플레이 기록을 시간순으로 확인할 수 있습니다.

### 4-7. 기본 퀴즈 수

기본 퀴즈는 총 8개이며, 평가표 요구사항인 5개 이상 조건을 충족합니다.

## 5. 코드 구조 설명

### `main.py`

프로그램 시작점입니다. `QuizGame` 객체를 만들고 `run()`을 호출합니다.

### `quiz.py`

퀴즈 한 문제를 표현하는 `Quiz` 클래스를 정의합니다.

담당 역할:

- 퀴즈 데이터 보관
- 생성 시 유효성 검사
- 문제 출력 문자열 생성
- 정답 확인
- 딕셔너리 변환과 복원

### `quiz_game.py`

게임 전체 흐름을 조율하는 `QuizGame` 클래스를 정의합니다.

담당 역할:

- 메뉴 출력
- 메뉴별 기능 분기
- 상태 저장 타이밍 결정
- 세션/카탈로그/저장소 모듈 연결
- 안전 종료 처리

### `console_io.py`

콘솔 입력과 공통 입력 검증, 안전 종료 예외를 담당합니다.

담당 역할:

- `prompt_text()`
- `prompt_int()`
- `SafeExitRequest`

### `state_store.py`

`state.json` 읽기/쓰기와 상태 검증을 담당합니다.

담당 역할:

- 파일 없음 처리
- 손상된 JSON 및 잘못된 스키마 복구
- 현재 스키마 기준 필수 키와 타입 검증
- 퀴즈/히스토리 복원
- `state.json` 키 상수 기반 저장/복원

### `quiz_session.py`

퀴즈 플레이 세션을 담당합니다.

담당 역할:

- 문제 수 선택
- 랜덤 출제
- 정답 입력과 힌트 처리
- 점수 계산
- 중단 시 부분 결과 반환

### `quiz_catalog.py`

퀴즈 추가, 목록, 삭제를 담당합니다.

### `scoreboard.py`

최고 점수와 히스토리 표시를 담당합니다.

### `game_constants.py`

점수, state 버전 같은 숫자 상수뿐 아니라 아래 문자열 상수도 함께 관리합니다.

- 메뉴 문구
- 입력 프롬프트
- 안내 메시지
- 점수/히스토리 표시 라벨
- `state.json` 키 이름

### `default_data.py`

기본 퀴즈 데이터와 기본 상태 생성 함수를 제공합니다.

## 6. state.json 데이터 파일 설명

### 파일 경로

- `./state.json`

### 인코딩

- UTF-8

### 역할

`state.json`은 프로그램의 전체 상태를 저장하는 파일입니다.

- 퀴즈 목록
- 다음 퀴즈 ID
- 최고 점수
- 플레이 히스토리

### 읽기 흐름

프로그램 시작 시 `QuizGame.__init__()`가 `StateStore`를 통해 상태를 불러옵니다.

- 파일이 있으면 JSON을 읽어 메모리에 반영합니다.
- 파일이 없으면 기본 퀴즈 데이터로 새 파일을 만듭니다.
- 파일이 손상되었거나 필수 키가 빠졌거나 타입/값 규칙이 맞지 않는 잘못된 스키마면 안내 메시지를 출력한 뒤 전체 state를 기본 데이터로 복구합니다.

### 쓰기 흐름

다음 시점에 `save_state()`가 호출됩니다.

- 퀴즈 추가 후
- 퀴즈 삭제 후
- 퀴즈 플레이 정상 종료 후
- 퀴즈 플레이 도중 입력 중단 후
- 프로그램 종료 시

현재 구현에서는 `version`, `next_quiz_id`, `best_score`, `quizzes`, `history` 같은 키 이름도 `game_constants.py`의 상수를 통해 일관되게 관리합니다.  
`history` 항목은 `played_at`, `selected_count`, `answered_count`, `correct_count`, `score`, `hint_used_count`를 모두 갖춘 현재 스키마만 정상 기록으로 취급하며, 숫자 문자열·`bool`·`float` 같은 암묵적 형변환 입력은 허용하지 않습니다.

### 현재 스키마

```json
{
  "version": 1,
  "next_quiz_id": 9,
  "best_score": 0,
  "quizzes": [
    {
      "id": 1,
      "question": "컴퓨터의 두뇌라고 자주 불리는 부품은 무엇일까요?",
      "choices": ["RAM", "CPU", "SSD", "GPU"],
      "answer": 2,
      "hint": "명령을 실행하고 계산을 수행하는 부품입니다."
    }
  ],
  "history": [
    {
      "played_at": "2026년 04월 13일 20시 39분 03초",
      "selected_count": 2,
      "answered_count": 1,
      "correct_count": 1,
      "score": 10,
      "hint_used_count": 0
    }
  ]
}
```

`answered_count`는 최종적으로 유효한 답 입력까지 완료된 문제 수를 뜻합니다.  
현재 코드는 `history`를 예전 형식까지 확장 해석하지 않고, 위 구조 그대로 엄격하게 검증해서 읽습니다. 숫자 문자열이나 `bool`처럼 자동 변환될 수 있는 값도 정상 데이터로 받아들이지 않습니다.

## 7. 파일 구조

현재 구조는 "원격 저장소에 올라가는 Python 코드"와 "실행하면서 생기는 파일"을 구분해서 보는 편이 더 이해하기 쉽습니다.

### 저장소 기준 구조

```text
task2/
├── README.md
├── main.py
├── quiz.py
├── quiz_game.py
├── quiz_session.py
├── quiz_catalog.py
├── state_store.py
├── scoreboard.py
├── console_io.py
├── game_constants.py
└── default_data.py
```

### 파일 역할 한눈에 보기

- `main.py`: 프로그램 시작점
- `quiz.py`: 퀴즈 1문제를 표현하는 데이터 모델
- `quiz_game.py`: 전체 게임 흐름 조율
- `quiz_session.py`: 퀴즈 플레이 진행, 힌트, 채점
- `quiz_catalog.py`: 퀴즈 추가, 목록, 삭제
- `state_store.py`: `state.json` 읽기/쓰기, 스키마 검증, 복구
- `scoreboard.py`: 최고 점수와 플레이 기록 출력
- `console_io.py`: 공통 입력 검증과 안전 종료 처리
- `game_constants.py`: 숫자 상수, 안내 문구, JSON 키 이름 관리
- `default_data.py`: 기본 퀴즈 데이터와 초기 상태 생성

### 실행 시 생성되는 파일

```text
task2/
└── state.json
```

- `state.json`: 프로그램 실행 후 생성되는 저장 파일
- 저장 내용: 퀴즈 목록, 다음 퀴즈 ID, 최고 점수, 플레이 히스토리
